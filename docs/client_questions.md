# 구현 착수 전 확인 질문서 (필수)

아래 질문에 답해주시면, 바로 실제 수집 코드(Playwright)와 보고서를 완성할 수 있습니다.

## A. 로그인/접근
1. `관계자` 버튼 클릭 후 로그인 폼이 **팝업(모달)** 인가요, **별도 페이지 이동**인가요?
2. 로그인 실패 메시지 형태는 무엇인가요?
3. CAPTCHA/OTP/추가 인증이 전혀 없나요?
4. 계정 잠금 정책(실패 횟수 제한)이 있나요?

## B. 데이터 화면 구조
5. 고용보험 6개 지표가 항상 동일한 순서/이름으로 보이나요?
6. 전국/17개 시도/경기 31개 시군 전환은 어떤 UI(탭/드롭다운/버튼)인가요?
7. 현재/2개월 전 값은 항상 동시에 보이나요, hover/클릭해야 보이나요?
8. 숫자 단위(명/개/%)는 지표별로 고정인가요?

## C. 보고서 확정
9. 보고서는 월별 1개 화면이면 충분한가요, 요약+상세 2~3개 페이지가 필요한가요?
10. 우선순위 정렬 기준은 `위기 > 관심 > 정상`으로 고정할까요?
11. 비교 기준은 `2개월 전`만 사용할까요, 전월 비교도 추가할까요?
12. CSV 다운로드 외에 Excel/PDF도 필요한가요?

## D. 운영/예외
13. 월말이 휴일이면 당일 실행 vs 익영업일 실행 중 어떤 정책으로 할까요?
14. 수집 실패 시 재시도 횟수(예: 2회)와 실패 알림(메일/슬랙)이 필요할까요?
15. 데이터 정정(사이트 원본 수정) 시 과거 스냅샷 덮어쓰기 허용할까요?

## E. 검증 데이터
16. 개발 중 사용할 테스트 계정을 제공할 수 있나요?
17. 샘플 월(예: 최근 1개월)의 정답 데이터(수작업 캡처/엑셀)가 있나요?

- A1: 모달
- A2: 빨간 글씨 "아이디 또는 비밀번호 오류"
- B6: 좌측 지역탭 + 상단 드롭다운
- C9: 요약 1페이지 + 상세 1페이지
- D13: 휴일이면 익영업일
